# Lazy Auto Ops - 服务器部署指南（x86 Linux）

## 📋 部署概览

**开发环境**: ARM Mac  
**目标服务器**: x86 Linux  
**部署方式**: Docker  
**预计时间**: 15-20分钟

---

## 🎯 部署流程（5步）

### 步骤1：本地准备（在Mac上）

#### 1.1 修复 topology/handler.go 文件（必须！）

```bash
# 用文本编辑器打开 topology_handler_code.txt
# 复制全部内容到 plugins/topology/handler.go
# 保存文件

# 验证文件
ls -lh plugins/topology/handler.go  # 应该 > 5KB
file plugins/topology/handler.go     # 应该显示 "Go source"
```

#### 1.2 打包项目

```bash
cd lazy-auto-ops

# 创建部署包（排除不必要的文件）
tar -czf lazy-auto-ops.tar.gz \
  --exclude='bin' \
  --exclude='data/*.db' \
  --exclude='.git' \
  --exclude='node_modules' \
  .

# 查看打包结果
ls -lh lazy-auto-ops.tar.gz
```

---

### 步骤2：上传到服务器

#### 2.1 使用 scp 上传

```bash
# 替换为你的服务器信息
SERVER_IP="your-server-ip"
SERVER_USER="root"

# 上传文件
scp lazy-auto-ops.tar.gz ${SERVER_USER}@${SERVER_IP}:/root/

# 或者使用密钥
scp -i ~/.ssh/your-key.pem lazy-auto-ops.tar.gz ${SERVER_USER}@${SERVER_IP}:/root/
```

#### 2.2 使用 rsync 上传（推荐，支持断点续传）

```bash
rsync -avz --progress \
  --exclude='bin' \
  --exclude='data/*.db' \
  --exclude='.git' \
  lazy-auto-ops/ ${SERVER_USER}@${SERVER_IP}:/root/lazy-auto-ops/
```

---

### 步骤3：服务器环境准备

#### 3.1 SSH登录服务器

```bash
ssh ${SERVER_USER}@${SERVER_IP}
# 或使用密钥
ssh -i ~/.ssh/your-key.pem ${SERVER_USER}@${SERVER_IP}
```

#### 3.2 安装Docker（如果未安装）

```bash
# CentOS/RHEL
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
sudo systemctl start docker
sudo systemctl enable docker

# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
docker --version
docker compose version
```

#### 3.3 配置Docker（可选但推荐）

```bash
# 配置国内镜像加速（可选）
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.mirrors.ustc.edu.cn",
    "https://hub-mirror.c.163.com"
  ]
}
EOF

sudo systemctl daemon-reload
sudo systemctl restart docker
```

---

### 步骤4：解压和配置

#### 4.1 解压项目

```bash
cd /root
tar -xzf lazy-auto-ops.tar.gz
cd lazy-auto-ops

# 或者如果使用rsync，直接进入目录
cd /root/lazy-auto-ops
```

#### 4.2 验证文件

```bash
# 检查 topology/handler.go
ls -lh plugins/topology/handler.go
# 应该显示 > 5KB

# 如果文件为空，需要手动修复
# 可以从本地再次上传该文件
```

#### 4.3 配置文件（可选）

```bash
# 编辑配置文件
cd deploy/docker
cp config.yaml config.yaml.bak
vim config.yaml

# 修改需要的配置，如：
# - 数据库配置
# - JWT密钥
# - 插件配置
# - AI API Key等
```

---

### 步骤5：Docker部署

#### 方式A：使用 Docker Compose（推荐）

```bash
cd /root/lazy-auto-ops/deploy/docker

# 启动服务
docker compose up -d --build

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f
```

#### 方式B：使用部署脚本

```bash
cd /root/lazy-auto-ops

# 给脚本执行权限
chmod +x docker-deploy.sh

# 运行脚本（选择方式2）
./docker-deploy.sh
```

#### 方式C：Docker直接运行

```bash
cd /root/lazy-auto-ops

# 构建镜像
docker build -t lazy-auto-ops:latest .

# 创建数据目录
mkdir -p data

# 运行容器
docker run -d \
  --name lazy-auto-ops \
  --restart unless-stopped \
  -p 8080:8080 \
  -v $(pwd)/data:/app/data \
  -v $(pwd)/configs/config.yaml:/app/configs/config.yaml:ro \
  -e TZ=Asia/Shanghai \
  lazy-auto-ops:latest

# 查看日志
docker logs -f lazy-auto-ops
```

## 🔧 配置防火墙

### CentOS/RHEL (firewalld)

```bash
# 开放8080端口
sudo firewall-cmd --permanent --add-port=8080/tcp
sudo firewall-cmd --reload

# 查看开放的端口
sudo firewall-cmd --list-ports
```

### Ubuntu/Debian (ufw)

```bash
# 开放8080端口
sudo ufw allow 8080/tcp
sudo ufw reload

# 查看状态
sudo ufw status
```

### 云服务器安全组

如果使用阿里云、腾讯云等，还需要在控制台配置安全组规则：
- 协议：TCP
- 端口：8080
- 源地址：0.0.0.0/0（或指定IP）

## 🚀 生产环境部署（推荐）

### 1. 使用生产环境配置

```bash
cd /root/lazy-auto-ops/deploy/docker

# 创建环境变量文件
cp .env.example .env
vim .env

# 修改密码（重要！）
# MYSQL_ROOT_PASSWORD=生成的强密码
# MYSQL_PASSWORD=生成的强密码
# REDIS_PASSWORD=生成的强密码
# JWT_SECRET=生成的强密钥

# 生成随机密码
openssl rand -base64 32
```

### 2. 启动生产环境

```bash
# 启动完整服务（MySQL + Redis + Nginx）
docker compose -f docker-compose.prod.yml up -d --build

# 查看状态
docker compose -f docker-compose.prod.yml ps

# 查看日志
docker compose -f docker-compose.prod.yml logs -f
```

### 3. 配置Nginx（可选）

```bash
# 如果需要HTTPS，准备SSL证书
mkdir -p ssl
# 上传证书文件到 ssl/ 目录

# 编辑nginx配置
vim nginx.conf
# 取消SSL配置的注释

# 重启nginx
docker compose -f docker-compose.prod.yml restart nginx
```

## 🎯 完整部署脚本（一键部署）

创建一个自动化部署脚本：

```bash
# 在服务器上创建 deploy.sh
cat > /root/deploy.sh << 'EOF'
#!/bin/bash
set -e

echo "开始部署 Lazy Auto Ops..."

# 检查Docker
if ! command -v docker &> /dev/null; then
    echo "安装Docker..."
    curl -fsSL https://get.docker.com | sh
    systemctl start docker
    systemctl enable docker
fi

# 进入项目目录
cd /root/lazy-auto-ops

# 检查文件
if [ ! -f "plugins/topology/handler.go" ] || [ $(wc -c < plugins/topology/handler.go) -lt 1000 ]; then
    echo "错误: topology/handler.go 文件不正确"
    exit 1
fi

# 部署
cd deploy/docker
docker compose down
docker compose up -d --build

# 等待启动
echo "等待服务启动..."
sleep 10

# 健康检查
if curl -s http://localhost:8080/health | grep -q "ok"; then
    echo "✅ 部署成功！"
    echo "访问地址: http://$(hostname -I | awk '{print $1}'):8080"
    echo "默认账号: admin / admin123"
else
    echo "❌ 部署失败，请查看日志"
    docker compose logs
fi
EOF

chmod +x /root/deploy.sh
```

使用脚本：
```bash
/root/deploy.sh
```
