# 定时任务调度器实现方案

## 概述
为Lazy Auto Ops平台实现完整的Cron定时任务调度功能，支持Shell、Python脚本的定时执行和管理。

## 已完成功能

### 1. 调度器核心 ✅
**文件**: `plugins/task/scheduler.go`

基于 `robfig/cron/v3` 实现的高性能任务调度器：

- **任务加载**: 启动时自动加载所有启用的定时任务
- **动态管理**: 支持运行时添加/删除/更新任务
- **定期重载**: 每5分钟自动重载任务配置
- **并发执行**: 任务异步执行，互不阻塞
- **超时控制**: 支持任务执行超时设置
- **状态跟踪**: 实时记录任务执行状态

### 2. 核心功能

#### 任务调度
```go
type Scheduler struct {
    db       *gorm.DB
    cron     *cron.Cron
    tasks    map[string]cron.EntryID
    mu       sync.RWMutex
    ctx      context.Context
    cancel   context.CancelFunc
}
```

**功能特性**:
- 支持标准5字段和带秒的6字段Cron表达式
- 自动计算下次执行时间
- 任务执行记录持久化
- 优雅启动和停止

#### 任务执行
- **Shell任务**: 执行Shell脚本和命令
- **Python任务**: 执行Python脚本
- **超时控制**: 可配置执行超时时间
- **输出捕获**: 记录标准输出和错误输出
- **状态管理**: 运行中/成功/失败/超时

### 3. 数据模型 ✅
**文件**: `plugins/task/models.go`

```go
type Task struct {
    Name        string     // 任务名称
    Type        string     // 类型: shell, python, ansible
    Content     string     // 脚本内容
    Targets     string     // 目标主机
    Cron        string     // Cron表达式
    Timeout     int        // 超时时间(秒)
    Enabled     bool       // 是否启用
    LastRunAt   *time.Time // 最后执行时间
    NextRunAt   *time.Time // 下次执行时间
    CreatedBy   string     // 创建人
}

type TaskExecution struct {
    TaskID    string     // 任务ID
    TaskName  string     // 任务名称
    Status    int        // 状态: 0运行中 1成功 2失败 3超时
    Output    string     // 输出内容
    Error     string     // 错误信息
    StartAt   time.Time  // 开始时间
    EndAt     *time.Time // 结束时间
    Duration  int        // 执行时长(秒)
    Executor  string     // 执行者
}
```

### 4. API接口 ✅
**文件**: `plugins/task/handler.go`, `plugins/task/plugin.go`

- `GET /api/task/tasks` - 任务列表
- `POST /api/task/tasks` - 创建任务
- `GET /api/task/tasks/:id` - 获取任务详情
- `PUT /api/task/tasks/:id` - 更新任务
- `DELETE /api/task/tasks/:id` - 删除任务
- `POST /api/task/tasks/:id/run` - 手动执行任务
- `POST /api/task/tasks/:id/enable` - 启用任务
- `POST /api/task/tasks/:id/disable` - 禁用任务
- `GET /api/task/executions` - 执行记录列表
- `GET /api/task/executions/:id` - 执行记录详情

## 使用示例

### 1. 创建定时任务

```bash
# 创建每天凌晨2点执行的备份任务
curl -X POST http://localhost:8080/api/task/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <token>" \
  -d '{
    "name": "数据库备份",
    "type": "shell",
    "content": "#!/bin/bash\nmysqldump -u root -p database > /backup/db_$(date +%Y%m%d).sql",
    "cron": "0 0 2 * * *",
    "timeout": 600,
    "enabled": true
  }'
```

### 2. Cron表达式格式

支持两种格式：

**标准5字段** (分 时 日 月 周):
```
0 2 * * *        # 每天凌晨2点
*/5 * * * *      # 每5分钟
0 0 * * 0        # 每周日凌晨
0 0 1 * *        # 每月1号凌晨
```

**带秒6字段** (秒 分 时 日 月 周):
```
0 0 2 * * *      # 每天凌晨2点
*/30 * * * * *   # 每30秒
0 */5 * * * *    # 每5分钟
```

### 3. 手动执行任务

```bash
# 立即执行任务
curl -X POST http://localhost:8080/api/task/tasks/{id}/run \
  -H "Authorization: Bearer <token>"
```

### 4. 查询执行记录

```bash
# 获取所有执行记录
curl http://localhost:8080/api/task/executions \
  -H "Authorization: Bearer <token>"

# 按任务ID筛选
curl http://localhost:8080/api/task/executions?task_id={task_id} \
  -H "Authorization: Bearer <token>"
```

### 5. 启用/禁用任务

```bash
# 启用任务
curl -X POST http://localhost:8080/api/task/tasks/{id}/enable \
  -H "Authorization: Bearer <token>"

# 禁用任务
curl -X POST http://localhost:8080/api/task/tasks/{id}/disable \
  -H "Authorization: Bearer <token>"
```

## 调度器生命周期

```
启动 → 加载任务 → 启动Cron → 定期重载 → 优雅停止
  ↓        ↓          ↓          ↓          ↓
初始化   验证配置   执行任务   更新配置   清理资源
```

### 启动流程
1. 创建调度器实例
2. 从数据库加载所有启用的定时任务
3. 验证Cron表达式
4. 添加任务到Cron调度器
5. 启动Cron引擎
6. 启动定期重载goroutine

### 执行流程
1. Cron触发任务
2. 创建执行记录（状态：运行中）
3. 异步执行任务脚本
4. 捕获输出和错误
5. 更新执行记录（状态：成功/失败/超时）
6. 更新任务的最后执行时间

### 停止流程
1. 取消上下文
2. 停止Cron引擎
3. 等待所有任务完成
4. 清理资源

## 安全考虑

1. **权限控制**: 所有API需要认证授权
2. **脚本审核**: 建议对脚本内容进行审核
3. **资源限制**: 设置合理的超时时间
4. **日志审计**: 记录所有操作和执行
5. **隔离执行**: 任务在独立进程中执行

## 性能优化

1. **异步执行**: 任务在goroutine中异步执行
2. **超时控制**: 使用context控制执行时间
3. **定期重载**: 5分钟重载一次，平衡实时性和性能
4. **索引优化**: 数据库字段添加索引
5. **记录限制**: 执行记录查询限制100条
