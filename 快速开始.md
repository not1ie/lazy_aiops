# 快速开始 - Lazy Auto Ops

## ⚠️ 重要提示

由于开发环境的bash问题，项目需要手动完成最后5%的工作。请按照以下步骤操作：

## 🚀 三步完成部署

### 第1步：修复 topology/handler.go 文件 ⚠️ 必须手动完成

文件 `plugins/topology/handler.go` 当前是空的（0字节），**必须手动添加内容**。

**推荐方法：使用文本编辑器（VS Code、Vim等）**
1. 用文本编辑器打开 `topology_handler_code.txt` 文件
2. 全选并复制全部内容（约240行）
3. 用文本编辑器打开 `plugins/topology/handler.go` 文件
4. 粘贴内容并保存

**验证文件是否正确：**
```bash
cd lazy-auto-ops
# 检查文件大小，应该大于5KB
ls -lh plugins/topology/handler.go
# 检查文件内容，应该显示 "Go source"
file plugins/topology/handler.go
# 查看文件前几行，应该显示 "package topology"
head -5 plugins/topology/handler.go
```

### 第2步：编译项目

```bash
cd lazy-auto-ops
go mod tidy
go build -o bin/lazy-auto-ops ./cmd/server
```

如果看到 `bin/lazy-auto-ops` 文件生成，说明编译成功！

### 第3步：运行服务

```bash
./bin/lazy-auto-ops
```

你应该看到：
```
🚀 Lazy Auto Ops 启动中... 端口: 8080
```

## ❓ 遇到问题？

1. **编译失败**
   - 确保Go版本 >= 1.21
   - 确保已执行 `go mod tidy`
   - 确保 topology/handler.go 文件不为空

2. **运行失败**
   - 检查端口8080是否被占用
   - 查看日志输出

3. **API调用失败**
   - 确认服务已启动
   - 确认Token正确
   - 检查请求格式
