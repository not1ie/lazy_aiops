# Lazy AIOps 回归验证清单

> 目标：发布前快速确认“功能可用 + UI 可用 + 安全策略可用”。

## 0. 基础环境

- 访问地址：`http://<host>:8080`
- 预置数据：至少 1 台 Docker 主机、1 个 K8s 集群、1 个 CMDB 主机
- 建议浏览器：Chrome 最新版

## 1. 启动与健康

| 编号 | 验证项 | 操作 | 预期结果 |
|---|---|---|---|
| H1 | 健康检查 | `curl -s http://127.0.0.1:8080/health` | 返回 `{"status":"ok"}` |
| H2 | 插件加载 | 查看服务日志 | 出现 `topology/system/ansible/docker/k8s/ai` 插件加载成功日志 |
| H3 | 版本一致性 | `docker image inspect lazy-aiops:<commit>` | `org.opencontainers.image.revision` 与当前代码 commit 一致 |

## 2. 登录与权限

| 编号 | 验证项 | 操作 | 预期结果 |
|---|---|---|---|
| A1 | 登录页信息 | 打开登录页 | 不展示默认账号/密码文案 |
| A2 | 默认密码策略 | 使用 `admin/admin123` 登录 | 进入“强制修改密码”弹窗，未改密前受限 |
| A3 | 改密放行路径 | 在弹窗中修改密码 | 修改成功并可重新登录 |
| A4 | 鉴权拦截 | 默认密码状态下访问业务接口 | 返回 428 或前端提示“先修改密码” |

## 3. Docker 模块

| 编号 | 验证项 | 操作 | 预期结果 |
|---|---|---|---|
| D1 | 列表渲染 | 打开“容器管理” | 列表正常，状态/资源显示正确 |
| D2 | 日志 | 点击“日志” | 能加载日志；失败时显示明确错误 |
| D3 | 执行命令 | 点击“执行命令”并执行 `echo ok` | 返回命令结果 |
| D4 | 终端 | 点击“终端” | 终端可连接；失败时显示预检错误 |
| D5 | 趋势 | 点击“趋势” | 图表正常，空数据时不报错 |
| D6 | UI 风格 | 检查 tab + 按钮 | 分段/胶囊风格一致，动画平滑 |

## 4. K8s 模块

| 编号 | 验证项 | 操作 | 预期结果 |
|---|---|---|---|
| K1 | 集群/命名空间联动 | 切换集群、命名空间 | 下拉联动正常 |
| K2 | Pod 终端 | 进入 K8s WebShell，点击连接 | 可连接；Pod 非 Running 时有明确报错 |
| K3 | 容器 shell 检测 | 选不存在 shell | 返回 shell 不存在类错误 |
| K4 | Deployments 信息 | 打开 Deployments | 展示关联域名/入口信息（如 ingress/host） |

## 5. 拓扑与 Ansible

| 编号 | 验证项 | 操作 | 预期结果 |
|---|---|---|---|
| T1 | 自动发现拓扑 | 在服务拓扑点击“自动发现” | 无 `UNIQUE constraint failed: service_nodes.name` |
| T2 | 多集群同名服务 | 不同 namespace/cluster 有同名服务 | 可并存，不冲突 |
| T3 | CMDB 同步 Inventory | 在 Ansible Inventory 点“从CMDB同步” | 同步成功并生成/更新 inventory |

## 6. AI 运维助手

| 编号 | 验证项 | 操作 | 预期结果 |

| AI1 | 资源选择 | 打开 AI 助手 | 可选择 Docker 与 K8s 目标 |
